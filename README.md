Project Overview :

Rankify is a robust, production-ready candidate ranking engine designed to act as an intelligent AI recruiter. Moving far beyond outdated, static Boolean keyword filtering, this platform utilizes high-dimensional dense embeddings to interpret the true semantic intent of complex job descriptions, seamlessly blending them with critical behavioral signals to deliver lightning-fast, high-precision candidate shortlists.

Core Architecture & Signal Fusion :

The engine processes candidate pools through a streamlined three-stage pipeline to achieve both contextual depth and execution speed. First, it maps job requirements and candidate profiles into a shared vector space using the all-MiniLM-L6-v2 semantic model, allowing it to evaluate conceptual alignment and transferrable skills over strict text matching. Second, a multi-signal integration layer aggregates and normalizes diverse operational data points, including technical assessment scores, platform activity, and career stability metrics like historical tenure. Finally, a harmonic fusion engine executes an automated mathematical weighting matrix to compute a final, unified composite score, sorting the entire candidate pool into an expertly ranked shortlist in milliseconds.

Repository Structure & Setup :

The repository is organized with a clean separation of concerns, featuring a frontend UI layout driven by src/App.tsx, src/main.tsx, and specialized elements within src/components/, while backend orchestration and data processing are managed by server.ts and ranking_engine.py. Global configuration, application package boundaries, and browser tab properties are configured via metadata.json, package.json, and index.html. To get started, developers simply need to install the core machine learning dependencies using the command pip install sentence-transformers pandas numpy to run the standalone ranking engine, followed by running npm install and npm run dev to launch the local Vite development server and interact with the live telemetry dashboard. Rebranding the application from its development placeholder is as simple as updating the logo text in the UI components, changing the HTML title tag, and updating the application name keys within the configuration JSON files.
